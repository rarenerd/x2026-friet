# ADHD: lyrics brainstorm — Friet van Desire

**Probleem:** De TL-Buis lyrics zijn lam en niet demoscene. We zoeken een
invalshoek voor Nederlandse teksten die wél cred hebben — grappig, grof,
technisch, meta — en passen op de bestaande Freed-from-Desire melodie.

**Herframing:** niet "betere patatgrappen" maar "wat zingt de scene nou
echt als er niemand kijkt".

## 2. Converge — shortlist

### ★ Top pick: "Nul bytes vrij — maar het moet smaller"
**Score: N8 V8 F9 = 8.25**

Waarom: Dit is de kern van de demoscene — vrijwillige zelfkastijding als
liefdestaal. Iedereen in de zaal herkent het gevoel van "het past niet,
maar het MOET passen." De metafoor is sterk genoeg voor een heel lied,
niet alleen een punchline. Bouwt voort op bestaande C64-optimalisatie-
cultuur (1K intro's, 64-byte cracktros, cycle-counting). Past perfect
bij een SID-track die zelf al tot 3 kanalen is gemutileerd.

**Hoe:**
- Elke couplet = een andere constraint war story (1K, zero page, cycle-count)
- Refrein: "nul bytes vrij maar het moet smaller / geen raster te veel, geen cycle te mager"
- Past op bestaande melodie — het refrein heeft exact het juiste aantal syllaben
- Ticker scrollt de regels synchroon met de SID playback

### 2e: "Ik heb geen leven / behalve raster interrupts"
**Score: N8 V8 F9 = 8.25**

Waarom: De twee regels die alles zeggen. Rauw, eerlijk, geen poespas.
Absoluut geen patat-metafoor te bekennen. De raster interrupt als de
enige constante in het leven van een scener. Risico: kan te nihilistisch
worden — heeft een opbeurende brug nodig.

### 3e: "NFO cult"
**Score: N8 V8 F8 = 8.0**

Waarom: Briljant concept, elke scener heeft duizenden NFO's gelezen.
De structuur (GROUP / TYPE / RELEASE / GREETINGS / NOTES) geeft vanzelf
een songstructuur. Risico: ongelijke syllabenaantallen per NFO-field
kunnen de melodie forceren.

### Traps (vermijden)
- Reset / lader blijft hangen — 1 goede regel, geen 48 bars. Uitgezakt na 8 maten.
- Patat-saus metafoor — te dicht bij huidige lamme thema. Betere verpakking, zelfde probleem.
- IRQ chain — conceptueel sterk, praktisch onuitvoerbaar in onze interrupt-architectuur.
- Kijk naar de swirl — te simpel, houdt geen heel nummer.

## 3. Focus — verdiepingen

### A. ★ "Nul bytes vrij — maar het moet smaller" (diepste uitwerking)

**Schets:**
De bestaande melodie en syllable-timing uit `docs/song_layers.yaml` (T7
vocal track) blijft onaangetast. De ticker in de .prg leest lyric-events
als (start_beat, end_beat, text_ptr) triplets — Dutch lyrics worden
een nieuwe data block + triplet table, geen engine-wijziging.

Elke couplet map op een constraint war story:
- **Couplet 1:** 1K intro — "tweeduizend bytes? luxe / ik doe het in een K / scroller, SID, effecten / en nog een beetje vrije[sic] tijd"
- **Couplet 2:** Zero page — "acht pagina's nul / meer heb ik niet nodig / pointertje hier, pointertje daar / de raster geeft me hoop"
- **Couplet 3:** Cycle-counting — "drie cycles voor een staartje / een sprite in de weg / één cycle over? nou mooi / dan doe ik er een effect bij"
- **Refrein (vast):** "nul bytes vrij maar het moet smaller / geen raster te veel, geen cycle te mager / ik schraaf tot het bloedt en dan nog een keer / tot de SID gilt: 'laat me met rust'"

**Load-bearing risk:**
Syllabische alignering. Elke Nederlandse regel moet exact passen in de
bestaande T7-noot-duren. Als "tweeduizend" 3 syllaben heeft maar het
origineel 4 noot-onsets, dan valt de ticker uit sync. Oplossing: maak
een syllable_budget.py script dat uit song_layers.yaml de noot-duren
per bar haalt en een template geeft.

**Eerste stap:**
`scripts/syllable_budget.py` — leest T7 note-onsets uit song_layers.yaml,
per bar de noot-duren in 16e-noten, plus het Engelse syllable count.
Output: een tabel per sectie met "syllable budget per note".

**Sub-ideeën:**
1. Dual-language remix — Engels in couplet 1, Nederlands in couplet 2-3
2. Ticker-responsive border glitch — bij "cycle te mager" flikkert de border
3. Outro: SID valt stil, ticker toont "NUL BYTES VRIJ" — fade naar zwart

### B. "Ik heb geen leven / behalve raster interrupts"

**Schets:**
Niet één verhaal, maar vignetten. Elk couplet is een scène uit het
demo-leven:
- **Couplet 1:** Verjaardagsdiner gemist omdat de IRQ niet werkte
- **Couplet 2:** Relatie mislukt — "ze snapte niet waarom 'de raster staat in $D012' een geldig excuus was"
- **Couplet 3:** Party — de hele tijd in een hoekje zitten coderen terwijl de muziek speelt
- **Refrein:** "Ik heb geen leven / behalve raster interrupts / en als die hapert / heb ik helemaal niks"
- **Brug:** De raster interrupt valt uit (glitch) — even is er stilte. Dan komt het refrein harder terug.

**Load-bearing risk:**
Zelfde als A — syllable fitting. De raster interrupt is een 3-lettergrepig
woord (ras-ter-in-ter-rupt) dat in het refrein moet passen.

**Eerste stap:**
Schrijf het refrein in het Nederlands en test tegen de noot-onsets van
het originele "Freed from Desire" refrein. Als het past, is de rest
invulbaar.

**Sub-ideeën:**
1. Raster-balken als visual — de ticker-regels zijn gekleurde balken, zichtbaar IRQ-gedreven
2. Turkse Pizza verse — de echte party-beleving onder de raster interrupt
3. Outro: interrupt stopt — "..." — fade

### C. "NFO cult"

**Schets:**
Letterlijk een NFO-bestand als lyric. De structuur van een NFO geeft
vanzelf een songstructuur:
- **Intro:** ASCII-box: "▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄" — gezongen als "la la la"
- **Couplet 1:** "GROUP......: deFEEST / TYPE.......: SONG  / RELEASE....: X2026"
- **Refrein:** "▄▄ FRIET VAN DESIRE ▄▄ / ▄▄ deFEEST 2026 ▄▄"
- **Couplet 2:** "GREETINGS..: Anus / Kloot / Augurk / deFEEST / X2026"
- **Bridge:** "NOTES......: 6581 only / 3 voices / zero empathy / 175 BPM / your CPU is crying"
- **Outro:** "▄▄ END OF NFO ▄▄"

**Load-bearing risk:**
Field-value paren geven ongelijke syllable counts. "TYPE.......: SONG"
is 16 chars maar "GREETINGS..: Anus" is langer. Truncation of melisma
kan onnatuurlijk klinken.

**Eerste stap:**
Neem de eerste 8 maten van de T7 melodie, schrijf de noot-onsets uit,
en schrijf één NFO-field-line die past zonder syncopatie.

**Sub-ideeën:**
1. Visual: ticker typt elke regel uit alsof het een terminal is
2. Koorparticipatie — "▄▄ sing along! ▄▄" in de ticker
3. PETSCII-to-NFO charset swap voor ASCII-art

## 4. Provocatie

Wat als de lyrics *met de SID mee-veranderen*? Niet één vaste tekst,
maar een systeem dat op elke playthrough een andere combinatie van
constraint-war-stories kiest uit een pool — zodat de ticker nooit
hetzelfde lied twee keer hetzelfde laat zien. Elke luisterbeurt is
een andere demo-scene anekdote. De SID speelt dezelfde noot, maar
de tekst zoekt een andere weg door dezelfde 3 kanalen.
